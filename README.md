# Push_swap

*This project has been created as part of the 42 curriculum by acirino and marcoppo.*

## Description
**Push_swap** is an algorithm project where the goal is to sort a set of integer values using two stacks and a limited set of operations. The program calculates and outputs the smallest sequence of instructions required to sort the data, focusing on **algorithmic complexity** (Big-O notation) and performance optimization.
### Bonus Feature: Checker Program
The project includes a **checker** program as a bonus feature. Its purpose is to verify whether the list of operations generated by `push_swap` actually sorts the stack properly. 
- It takes the same stack of integers as an argument.
- It reads sorting instructions from the standard input.
- After executing the instructions, it displays **"OK"** if the stack is sorted and stack B is empty, or **"KO"** otherwise.
- It includes robust error management, displaying **"Error"** for invalid inputs or non-existent instructions.

## Instructions

### Compilation
The project includes a `Makefile` to handle compilation with the required flags (`-Wall -Wextra -Werror`):
```bash
make        # Compiles the push_swap program
make bonus  # Compiles the checker program
make clean  # Removes object files
make fclean # Removes object files and binaries
```
*Note: The compilation automatically manages the internal `libft` library.*

### Execution
To run the program, provide a list of integers as arguments:
```bash
./push_swap 2 1 3 6 5 8
```
**Bonus execution (Piping):**
To run the checker, combine the normal execution with a pipe and the same arguments as the main program
```bash
ARG="4 67 3 87 23"; ./push_swap $ARG | ./checker $ARG
```

The program supports specific strategy selectors and a benchmark mode:
- `--simple`: Forces the $O(n^2)$ algorithm.
- `--medium`: Forces the $O(n\sqrt{n})$ algorithm.
- `--complex`: Forces the $O(n \log n)$ algorithm.
- `--adaptive`: Uses the disorder-based selection (Default).
- `--bench`: Displays performance metrics (disorder, strategy used, operation count) to stderr.

## Algorithms & Technical Choices

This implementation features four distinct sorting strategies to handle different input configurations efficiently:

1.  **Simple Algorithm $O(n^2)$:** An optimized selection/insertion sort adaptation used for small datasets or low-disorder stacks.
2.  **Medium Algorithm $O(n\sqrt{n})$:** Uses a chunk-based sorting strategy, dividing the stack into blocks to minimize moves.
3.  **Complex Algorithm $O(n \log n)$:** Implements a **Radix Sort** adaptation (bitwise sorting) to handle large datasets within the operation limits.
4.  **Adaptive Algorithm:** Our primary strategy. It calculates the **disorder metric** before performing any moves.

### The Disorder Metric
As required by the subject, we calculate disorder as a value between 0 and 1 by counting "mistakes" (where a larger number appears before a smaller one) relative to the total possible pairs. 
**Thresholds used:**
- **Low disorder (< 0.2):** Simple Sort $O(n^2)$.
- **Medium disorder (0.2 ≤ d < 0.5):** Medium Sort $O(n\sqrt{n})$.
- **High disorder (≥ 0.5):** Radix Sort $O(n \log n)$.
